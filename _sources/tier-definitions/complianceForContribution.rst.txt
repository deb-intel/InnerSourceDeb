Compliance for Contribution
###########################

Compliant to corporate guidelines and is appropriate for reuse.


Tier Definitions
****************

.. _tier_complianceForContribution_start:

* **Tier 5.** No corporate compliance enforcement.
* **Tier 4.** Periodical manual review of new code for corporate compliance.
* **Tier 3.** Review of new code for corporate compliance upon release. Developer code review for well-written code. Internal products should meet AppSec (IAPM) requirements. External should meet SDL and SWLC requirements.
* **Tier 2.** Some applicable automated checks are integrated into pull-request (PR) validation pipeline.
* **Tier 1.** All applicable automated checks are integrated into pull-request (PR) validation pipeline. Examples: SDL (klockwork, snyk), SWLC (protex and IPLDT), etc.

.. _tier_complianceForContribution_end:
