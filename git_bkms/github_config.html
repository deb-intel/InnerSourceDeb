Prerequisites#

For for initial Git configuration, follow the Intel 1Source instructions below. Upon completion of these steps, continue the process below.

  • 1Source Onboard Process

  • 1Source Environment Setup

Use the 1Source Devtool for git configuration.

Note

The 1Source Devtool assists in creating a global .gitconfig file. Git uses a hierarchical configuration method. The home directory .gitconfig is global, whereas the .git/config in a repository root is local. The latter can override the former with repository-specific options.

Learn more Optional Git configuration.

SSH Configuration#

We recommend using SSH for connecting with GitHub, mainly for security reasons. Whereas both HTTPS and SHH protocols are acceptable, we prefer using SSH because its benefits outweigh the risks associated with using HTTPS. Using SSH also does not require entering a password for basic push/pull events.

First, follow the SSH instructions from 1Source:Connect to GitHub with SSH. Second, follow GitHub instructions on Checking for or generating SSH keys. For additional help, follow the multi-platform instructions below.

Verify or create an SSH key#

Follow these steps to verify or generate an SSH key. If you don't have an existing SSH key, you may also follow the GitHub instruction, Generating a new SSH key.

  1. Open a Command Line Interface (CLI) on your workstation.

  2. Enter a command to verify your workstation has an existing SSH key.

    ls -la ~/.ssh

    ls -la ~/.ssh

    ssh

    Note

    If using PowerShell and Windows 10 Fall 2018 build or later, the ssh client should be installed. A message like the one below should appear.

    usage: ssh [-46AaCfGgKkMNnqsTtVvXxYy] [-B bind_interface] [……………………………………………….]

  3. View the SSH public key, or verify which agent is used.

    cat ~/.ssh/id_rsa.pub

    cat ~/.ssh/id_rsa.pub

    For PowerShell on Windows 10 or later, follow these steps.

    Get-WmiObject win32_service | ?{$_.Name -like 'ssh-agent'} | select PathName

  4. If using PowerShell, follow these instructions to use OpenSSH and Make PowerShell Remember SSH PassPhrase.

Add SSH public key to GitHub Account#

  1. If you already added your SSH key to GitHub, skip the following section.

  2. On GitHub in upper right, select the drop-down menu below your profile.

  3. Select Settings.

  4. In the left-side menu, select SSH and GPG keys.

  5. On the page SSH keys, click the button New SSH key.

  6. Return to your CLI to view your public SSH key.

View local SSH public key#

cat ~/.ssh/is_rsa.pub

cat ~/.ssh/is_rsa.pub

Get-Content -Path $HOME\.ssh\id_rsa.pub | Set-Clipboard

  1. In Linux or Git Bash, you may enter the command to copy SSH public key clip < ~/.ssh/id_rsa.pub

    Note

    If the clip command doesn’t work, copy the contents of the id_rsa.pub from the previous step.

Add to GitHub SSH public key#

  1. Paste your SSH key into the “Key” field.

  2. In the Title field, enter a descriptive label, preferably one connected to your workstation or laptop.

  3. Select Add SSH key.

  4. If prompted, enter your GitHub password.

    See also: Adding a new SSH key to your GitHub account.

Optional Git configuration#

Following a few options for git config –global to help simplify your workflow and ease collaboration.

Set the editor. Some editor options include: atom, emacs, nano, etc.

git config --global core.editor nano

If using Windows, you can auto-convert LF to CRLF line-endings when you check out files.

git config --global core.autocrlf true

If using Linux* OS or macOS* and a file with CRLF is introduced, you tell Git to convert CRLF to LF on committing files.

git config --global core.autocrlf input

The above configuration can be useful when users collaborate across platforms like Windows and Linux.

For more details, see Customizing Git, “Formatting and Whitespace.”
